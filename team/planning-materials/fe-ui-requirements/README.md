# UI 기획안 디렉토리

PM Agent가 자동으로 생성하는 디렉토리입니다.
직접 파일을 작성할 경우 아래 규칙을 따르세요.

## 파일 네이밍 규칙

| 파일 | 형식 | 예시 |
|------|------|------|
| 텍스트 기획안 | `{티켓번호}-{화면명}-ui-spec.md` | `PROJ-123-login-ui-spec.md` |
| 와이어프레임 | `{티켓번호}-{화면명}-wireframe.html` | `PROJ-123-login-wireframe.html` |

## 와이어프레임 HTML 작성 규칙

PM Agent가 자동 생성하며, 아래 기준을 따른다.

**인터랙션 포함 여부 결정:**

| 상황 | HTML 유형 |
|------|----------|
| 단순 정보 표시, 레이아웃 확인만 필요 | 정적 HTML |
| 폼 제출 후 화면 전환 | 인터랙션 포함 |
| 성공/실패에 따라 다른 상태 표시 | 인터랙션 포함 |
| 모달, 토스트, 드로어 등 오버레이 | 인터랙션 포함 |
| 탭, 스텝, 위저드 등 단계 전환 | 인터랙션 포함 |

**HTML 규칙:**
- 스타일 없이 구조만 표현 (CSS 프레임워크 사용 금지)
- 인터랙션은 바닐라 JS로만 구현 (외부 라이브러리 금지)
- 각 상태를 `id="state-{name}"` div로 구분
- 숨겨진 초기 상태는 `style="display:none"` 으로 표시
- API 호출은 시뮬레이션으로 대체 (실제 fetch 금지)

## 권장 UI 기획안 (.md) 구조
```markdown
# {화면명} UI 기획안

## 화면 개요
- **목적**: ...
- **연결 API**: POST /api/v1/auth/login

## 화면 목록
- 로그인 폼 (기본 상태)
- 로그인 폼 (에러 상태)
- 메인 페이지 (로그인 성공 후)

## 유저 플로우
1. 로그인 폼 진입
2. 이메일/비밀번호 입력 후 로그인 버튼 클릭
   - 성공: 메인 페이지로 이동
   - 실패: 에러 메시지 표시, 폼 유지

## 컴포넌트 구성
### LoginForm
- 이메일 Input
- 비밀번호 Input
- 로그인 Button (로딩 상태 포함)
- 에러 메시지 영역 (실패 시 표시)
- 회원가입 링크 / 비밀번호 찾기 링크

## 상태 정의
- 로딩: 버튼 비활성화 + 로딩 스피너
- 실패: 에러 메시지 표시, 폼 유지
- 성공: 메인 페이지로 이동

## 엣지 케이스
- 이메일 형식 오류 → 클라이언트 유효성 검사
- 비밀번호 8자 미만 → 클라이언트 유효성 검사
- API 호출 중 → 버튼 비활성화
```
