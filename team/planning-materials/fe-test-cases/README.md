# 프론트엔드 테스트 케이스 디렉토리

UI 테스트 케이스를 `.md` 형식으로 작성하여 이 폴더에 배치하세요.

## 파일 네이밍 규칙
`{티켓번호}-{화면명}-test-cases.md`

## 권장 테스트 케이스 구조
```markdown
# {화면명} UI 테스트 케이스

## TC-FE-001: 유저 목록 정상 로딩
- **대상 컴포넌트**: UserListPage
- **Mock API**: GET /api/v1/users → 200 (유저 3명 반환)
- **기댓값**: 유저 이름 3개가 화면에 표시됨
- **케이스 유형**: Happy Path

## TC-FE-002: API 실패 시 에러 메시지
- **대상 컴포넌트**: UserListPage
- **Mock API**: GET /api/v1/users → 500
- **기댓값**: "데이터를 불러올 수 없습니다" 텍스트 표시
- **케이스 유형**: Error
```
